Understanding Semantic HTML: The Strong Foundation of the Web

HTML is the backbone of every web page. However, many beginners immediately use the <div> tag for everything, without realizing that HTML has many specific tags designed to give meaning to the content structure. This is called Semantic HTML.

Why is Semantic HTML Important?

Using Semantic HTML is not just about code aesthetics. It is a best practice that has a huge impact on:

  1. SEO (Search Engine Optimization): Google can understand your website's content more easily.
  2. Accessibility: Screen readers for visually impaired users can navigate the page better.
  3. Maintainability: The code becomes easier to read, manage, and collaborate on by other developers.

Examples of Non-Semantic vs Semantic Tags

Instead of writing <div class="header">, it is better to use the <header> tag. A proper modern HTML5 structure usually consists of <header> for titles and navigation, <main> the main content, inside which there is an <article> for the blog post, and <footer> for copyrights. 


  <header> <!-- Judul dan Navigasi --> </header>
  <main>
    <article> <!-- Isi Artikel --> </article>
  </main>
  <footer> <!-- Hak Cipta --> </footer>

Common Beginner Mistakes

One of the biggest mistakes is using headings haphazardly. Never skip from <h1> directly to <h3> just because you like the font size. The HTML rule requires a hierarchical and sequential heading order (<h1> followed by <h2>, then <h3>).


"The purpose of HTML is not to control visual appearance, but to provide structure and meaning to the content. Leave the styling to CSS."
